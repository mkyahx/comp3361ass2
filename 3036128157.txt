1.Description of Taggers
I implement three different Taggers to solve the NER problem.
1.1 BiLSTM Taggers
In NER task, single directional LSTM is weak on recognize the relationship of the pair words, so I introduce BiLSTM model.
1.1.1 Architecture
input sequence → Embedding → Dropout → BiLSTM → Dropout → linear layer → tag
   [B, L]        [B, L, E]  [B, L, E] [B, L, H][B, L, H]   [B, L, T]

Embedding layer converts the tokens into vectors. Dropout layer is used to avoid overfitting. BiLSTM layer grab memory from context. Linear layer project the hidden state into tag space.

1.2 Transformer Tagger
1.2.1 Architecture

input tokens → Embedding → Positional Encoding → Transformer Encoder → Linear layer → tag
   [B, L]      [B, L, D]      [B, L, D]              [B, L, D]            [B, L, T]

Embedding layer converts the tokens into vectors. PositionalEncoding layer inject positional information by sin and cos encoding. TansformerEncoder subtracts features by self-attention mechanism. Linear layer project the protraits in to tag.

1.3 DistilBERT Tagger
1.3.1 Architecture
input sequence → Tokenizer → DistilBERT → Dropout → linear layer → tag
                  WordPiece    [B,L,768]   [B,L,768]   [B,L,T]
Tokenizer is the DistilBertTokenizerFast, used to align original labels. DistilBERT model is with pre-trained parameters, a transformer encoder with 6 layers. Dropout layers adjust regularization. Linear layer project the output into tag space.

2.Experimental Settings and Hyperparameters
2.1 training
2.1.1 optimization
AdamW and Adam optimizers are introduced to accelerate the first several epoch of training
2.1.2 Reducing learning rate
Introduced ReduceLROnPlateau to avoid the model go too fast in the later stage, which is conductive to fine-tuning
2.1.3 Masking
I used -100 on ignored labels to ensure the loss is only calculated on valid tokens
2.1.4 weight of zero
Since the number of tag 0 is significantly more than others, I deduce the weight for predicting tag 0 to avoid model predict every tag to 0.

2.2 Hyperparameter tuning

4. Analysis and Discussion

4.1 Comparision among models
4.1.1 BiLSTM
BiLSTM model shows the power of sequence model and result in the best F1 score. This finding suggest that sequence model is efficient at capturing local patterns. The high recall indicates that the model is good at finding entities.
4.1.2 Transformer
Transformer architecture is lagged behind in all the three indicators. This may lead to the conclusion that a self-attention structure without pre-trained parameters is struggling with finding the optimal solution when the training data is relatively small.
4.1.3 DistilBert
DistilBert model gives the highest precision. It is more cautious and accurate when it did predict an entity. The lower F1 score may be attributed to the subword tokenization process.

4.2 Comparision among entities
By observing the score obtained on each entity tag, we can reveal the strength and weakness of the models.

4.2.1 Performance of entities
GPE, PERCENT, and NORP are performed well in all the models, While EVENT and WORK_OF_ART showed the lowest scores. That is to say the models excel at recognizing entity types that are structured, clearly defined, and have prominent contextual features, while they lack deep semantic reasoning and open-domain generalization capabilities.

4.2.2 The advantages of each model
I observed the certain tag that one model excel the others and thus conclude the advantages of each model

4.2.2.1 BiLSTM: The Sequential Pattern Expert
BiLSTM achieve the highest F1 and also the Recall on GPE, PERSON, Indicating that LSTM is extremely good at capturing local dependency and tagging patterns.

4.2.2.2 DistilBert: Semantic Precision specialist
DistilBert model has the highest precision on MONEY and EVENT within only 5 epoch. This fact shows that the pre-trained parameters imply the knowledge of deep contextual understanding.

4.2.2.3 Transformer: Quantitative Changes Lead to Qualitative Changes
Transformer wins on the Recall of LANGUAGE tag, a type that only appears 22 times in the training data. This indicates that after larger epoch number (60 for Transformer compare to 20 for LSTM and 5 for Bert), the model became specialist on searching certain words.
